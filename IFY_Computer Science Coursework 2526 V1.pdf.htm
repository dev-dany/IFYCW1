Coursework assessment includes content from Learning Outcomes LO4 and LO7 of the syllabus and may have overlapping content from other topics.

Teachers at the Study Centre will be responsible for marking coursework in accordance with the NCUK processes and policies. 

Context Choices

The coursework contains two projects – a Python project and a database project. You must complete both projects submitting a written report for both. 

A scenario has been provided to set each problem in a real-world context. 

The projects are a substantial piece of work which assesses a variety of different skills including the development and demonstration of computational thought processes and database design. The following guidance should be considered before learners embark on this assessment. 

 Students have access to appropriate database software and a programming language (which is non-trivial and has a substantial coded element).

 Centre software and resources allow students to fully solve the set problems sensibly within the constraints of resources available. 

 Involve all elements of the skills of analysis, design, development and evaluation.

 Students are equipped with the skills in screen-shotting evidence, cropping images for suitability and annotating evidence.

Project documentation structure

Candidates are encouraged to use the NCUK coursework template, which provides structured guidance on what to include at each stage of the report. The template is provided in PowerPoint format to support ease of use, particularly for adding screenshots, annotations, and labels. While this format is not industry standard for a report, it has been selected to simplify the creation process for learners and marking process for centres.

Centres may adopt an alternative structure if preferred, but it must still allow candidates to fully demonstrate all assessment requirements.

Independence

While teachers may provide general guidance and support, each candidate's submission should reflect their own individual work. The solution should be independently designed and implemented by the student. To ensure fairness and authenticity, whole-class teaching must avoid leading learners toward identical outcomes.

Assessment Criteria

Learners are expected to demonstrate their ability to design, develop, test and evaluate their solutions in each project. The problem must be solvable effectively in the chosen language, so it is important there is a good match between the requirements of the problem and the language that is written in Python.

Data Files  

A prebuilt 2D array is provided for the Python task (rooms.py) 

To maximise learning time and minimise repetitive data entry, NCUK has provided preprepared CSV files for students to import into their database structures. Some of these files intentionally do not include header rows, allowing students to define and apply their own field names as part of the implementation process.

Students are not required to delete, amend, or add to the data unless instructed to do so within a specific task.

NCUK also provide .png image files for incorporating into business reports and forms. All data and images within the datasets are AI-generated. Any resemblance to real individuals, companies, or organisations is purely coincidental.

Task B: Database Task  

Northbridge University Accommodation Services  Coursework Brief 2025-26

 

You should submit a written report of 1000 words approximately which covers the following tasks.

 

Background

The University Accommodation Services (UAS) team at Northbridge University in London manages all on-campus student housing. Each year, thousands of students apply for accommodation, and the department requires a reliable, efficient database system to manage room allocations, tenancy records, and maintenance requests.

Project Overview

You have been hired as a junior database developer to help UAS replace their outdated spreadsheet system with a relational database. This system must store data about students, halls of residence, rooms, tenancies, and maintenance issues.

 

Requirements

Data to be stored must include:

•        Students: Full name, student ID, e-mail and phone number, subject of study, year of study, and accessibility needs.

•        Halls of Residence: Hall name, location, and total capacity.

•        Rooms: Room number, type (e.g. single or en-suite), accessibility, and price per week.

•        Tenancies: Which student is assigned to which room and for which academic year.

•        Maintenance Requests: Student ID, room ID, issue description, date raised, status (Pending or Resolved), and assigned staff member (BMC or SJL).

Key Relationships

•        Each hall contains multiple rooms.

•        A student can only live in one room per academic year.

•        A room can only be assigned to one student per tenancy period.

•        A student may submit multiple maintenance requests for their room.

  

Data Files Provided

Your tutor will provide the following files:

•        students.csv, rooms.csv, halls.csv, tenancies.csv & maintenance Requests.csv

Note: Students and Maintenance files do not include field names
